# 问题记录
## 蜂鸣声响起的时候会掉帧
这是由于单线程同步执行导致 winsound.Beep 是阻塞的，导致每帧显示之后要等蜂鸣结束才能继续播放下一帧

** 解决方案 **

采用python的多线程方式解决了掉帧问题
## 当无线耳机电量低的时候会导致丢声
在测试的时候偶然发现第一天能听到声音第二天却听不见了，而且不用耳机的时候发现电脑外放也是有声音的

** 原因 **：由于蓝牙耳机的省电协议导致丢声，音频过于短暂导致在待机中的耳机激活之前声音已经停止，所以就会听不见

** 解决方案： ** 

可行的方案大概有以下三种：
* 方案1：参考电脑系统音量的提示音，
即使你的耳机进入了这个省电模式，
只要调节系统音量就能听到系统音量的提示音，
你会发现系统音量的提示音也很短暂，
但是由于优先级高所以能够强行拉起蓝牙链路然后再播放提示音。 所以可以采用类似的方法强制通过高优先级通道发送音频
* 方案2：播发“静音领航信号”，
在执行真正的报警之前，
先播放一个极短的、频率极高（人类听不见）或者完全静音的音频片段。
这相当于给耳机发了一个“起床闹钟”。
* 方案3：在初始化程序时，开启一个背景线程，循环播放一段完全静音的 MP3。

这三种方案中最合适的是第二种，因为第一种相当于增加了蓝牙耳机的耗电量；最后一种要一直播放背景音从性能上来说并不合适。所以采用第二种方案。
